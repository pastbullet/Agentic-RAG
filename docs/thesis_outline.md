《基于大语言模型的网络协议文档自动化解析与代码生成系统》

---

## 第一章 绪论（约 5000 字）

### 1.1 研究背景与意义
- 网络协议实现的现状：手动阅读 RFC/IEEE 文档 → 手写协议栈代码，成本高、出错率高
- 交换机/路由器协议栈开发的痛点：文档长（几百页）、字段多、状态机复杂
- LLM 在代码生成领域的进展，以及在协议理解方面的潜力

### 1.2 国内外研究现状
- 协议状态机提取：FlowFSM（prompt chaining）、PSMBench（NeurIPS 2025 benchmark）、SpecGPT（3GPP）
- 协议规范形式化：ICSE 2025（LLM-Aided Automatic Modeling）、FLAG（SVA 生成）
- 协议测试验证：iPanda（一致性测试）、NeTestLLM（多 agent 测试）、Eywa（模型驱动测试）
- LLM 代码生成与验证：LLM4PLC（ICSE 2024，工业控制代码生成）
- 现有工作的不足：大多只做单一环节，缺乏端到端闭环

### 1.3 研究内容与主要贡献
- 贡献 1：端到端的协议文档解析与代码生成流水线
- 贡献 2：基于多维相似度的协议对象智能合并算法
- 贡献 3：面向协议帧结构的 C 代码自动生成与验证方法
- 贡献 4：支持人在回路的渐进式提取优化机制

## 第七章 总结与展望（约 2000 字）

### 7.1 工作总结
- 实现了端到端的协议文档自动化解析与代码生成系统
- 在 RFC2PSM benchmark 上的评测结果
- 系统的实际应用价值

### 7.2 不足与局限
- 依赖 LLM 能力，不同模型效果差异
- 仅支持固定格式帧结构，不支持 TLV 等动态结构
- 生成代码为骨架代码，非完整可运行协议栈

### 7.3 未来工作
- 支持更多协议类型（3GPP、IEEE 802）
- 引入多模态理解（协议图表、状态转移图）
- 代码生成扩展到完整协议栈框架
